# 开发指南

## 一、项目概述

**项目名称：** 智慧物业管理系统（hm-admin）

**项目简介：** 面向园区/物业管理的综合型后台管理平台，涵盖企业管理、楼宇管理、停车计费、一卡通管理等业务模块。

---

## 三、环境配置

### 1. 环境要求

- Node.js >= 16
- pnpm >= 7

### 2. 安装依赖

```sh
pnpm install
```

### 3. 启动开发服务

```sh
pnpm dev
```

### 4. 打包构建

```sh
pnpm build
```

### 5. 代码检查

```sh
pnpm lint
```

### 6. 单元测试

```sh
pnpm test:unit
```

## 六、开发规范

### 1. 命名规范

| 类型         | 规范              | 示例                     |
| ------------ | ----------------- | ------------------------ |
| 文件/文件夹  | 驼峰命名          | `ParkEnterprise.vue`     |
| 组件名       | 大驼峰            | `CarRule`                |
| 变量/函数    | 小驼峰            | `getRuleList`            |
| API 接口函数 | 小驼峰 + Api 后缀 | `calculateCardApi`       |
| 类型定义     | 大驼峰            | `EnterpriseItem`         |
| CSS 类名     | 短横线分隔        | `search-container`       |

### 2. 文件组织规范

- 接口请求统一放在 `src/apis/` 目录
- 类型定义统一放在 `src/types/` 目录
- 页面组件放在 `src/views/` 按模块分目录
- 公共组件放在 `src/components/`

### 3. 代码风格

- 使用 `<script lang="ts" setup>` 组合式 API
- 响应式数据使用 `ref()` / `reactive()`
- 异步请求使用 `async/await`
- 表单验证使用 Element Plus 的 `FormRules`

## 七、常用开发模式

### 1. 标准列表页模式

```
搜索区域 → 操作按钮区域 → 表格区域 → 分页区域
```

### 2. 接口调用模式

```typescript
// 1. 在 src/apis/ 中定义接口
export const getXxxApi = (params: XxxParams) => {
  return httpInstance.get('/xxx/list', { params })
}

// 2. 在 src/types/ 中定义类型
export interface XxxItem {
  id: number
  name: string
}

// 3. 在页面中调用
const list = ref<XxxItem[]>([])
const getList = async () => {
  const res = await getXxxApi(params.value)
  list.value = res.data.rows
  total.value = res.data.total
}
onMounted(() => getList())
```

### 3. 弹窗表单模式

```typescript
// 控制弹窗显隐
const dialogVisible = ref(false)
// 表单数据
const form = ref({ ... })
// 表单验证规则
const rules: FormRules = { ... }
// 确认提交
const handleConfirm = async () => {
  await submitApi(form.value)
  dialogVisible.value = false
  getList() // 刷新列表
}
```
