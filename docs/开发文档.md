# 黑马智慧园区 - 后台管理系统

## 一、项目介绍

### 1.1 项目概述

**黑马智慧园区**是一个基于 Vue3 技术栈开发的后台管理系统，主要用于园区的日常运营管理，包括园区管理、行车管理、物业费管理、一体杆管理、系统管理等核心模块。

### 1.2 技术栈

| 技术         | 说明                                                   |
| ------------ | ------------------------------------------------------ |
| Vue 3        | 前端框架，使用 Composition API + `<script setup>` 语法 |
| TypeScript   | 类型安全，提升代码可维护性                             |
| Vite         | 构建工具，开发环境快速热更新                           |
| Vue Router   | 路由管理，支持动态路由和权限控制                       |
| Pinia        | 状态管理，替代 Vuex                                    |
| Element Plus | UI 组件库                                              |
| Axios        | HTTP 请求封装                                          |
| ECharts      | 数据可视化图表                                         |

## 二、核心功能模块

### 2.1 登录模块

> **面试题：说一下你项目中的登录流程？**

1. 用户输入账号密码，点击登录
2. 调用登录接口，后端返回 `token`
3. 使用 Pinia 存储 `token` 和用户信息，并持久化到 `localStorage`
4. 登录成功后，通过 `router.push` 跳转到首页
5. Axios 请求拦截器中统一携带 `Authorization: Bearer {token}` 请求头
6. 响应拦截器中判断 `401` 状态码，自动跳转到登录页

**关键代码思路：**

```typescript
// 请求拦截器 - 统一添加 token
request.interceptors.request.use(config => {
  const token = useUserStore().token
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截器 - token 过期处理
request.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      useUserStore().clearUserInfo()
      router.push('/login')
    }
    return Promise.reject(error)
  }
)
```

### 2.2 权限控制

> **面试题：你项目中是怎么做权限控制的？**

#### 路由权限

1. 登录后请求用户权限信息，获取该用户可访问的路由列表
2. 使用 `router.addRoute()` 动态添加路由
3. 通过路由守卫 `beforeEach` 判断是否有 token，没有则跳转登录页

#### 按钮权限

1. 后端返回用户的按钮权限标识列表（如 `['sys:role:add', 'sys:role:del']`）
2. 通过自定义指令 `v-permission` 或在模板中使用 `v-if` 控制按钮显示隐藏

### 2.3 系统管理 - 角色管理

> **面试题：你做过最复杂的功能是什么？**

角色管理是系统管理模块的核心功能，包含角色列表、新增角色、编辑角色、删除角色、权限分配。

#### 2.3.1 角色列表页（SystemRole.vue）

**功能点：**

- 左侧角色列表展示，点击切换显示角色详情
- 右侧展示两个 Tab：功能权限（树形结构）、成员列表
- 支持新增、编辑、删除角色

**关键实现：**

```typescript
// 默认选中第一个角色
onMounted(async () => {
  await getRolesList()
  if (roleList.value.length > 0) {
    activeIndex.value = 0
    const roleId = roleList.value[0].roleId
    if (roleId !== undefined) {
      getRoleDetail(roleId)
      getRoleUserList(roleId)
    }
  }
})

// 删除角色 - 二次确认
const handleDeleteRole = (roleId: number) => {
  ElMessageBox.confirm('确认删除该角色吗？', '提示', {
    confirmButtonText: '确定',
    cancelButtonText: '取消',
    type: 'warning'
  }).then(async () => {
    const res = await delRoleUserAPI(roleId.toString())
    if (res.code === 10000) {
      ElMessage.success('删除成功')
      await getRolesList()
      // 删除后自动选中第一个角色
      getRoleDetail(roleList.value[0].roleId!)
    }
  })
}
```

#### 2.3.2 新增/编辑角色页（AddRole.vue）

**功能点：**

- 分步表单：第一步角色信息 → 第二步权限配置 → 第三步检查确认
- 新增和编辑复用同一个页面，通过 URL 的 `query` 参数区分
- 编辑模式下自动回填角色信息和权限树选中状态

> **面试题：新增和编辑怎么复用同一个页面？**

**核心思路：** 通过 `route.query.roleId` 判断是新增还是编辑模式。

```typescript
const route = useRoute()
const roleId = ref(route.query.roleId as string | undefined)
const isEdit = ref(!!roleId.value) // 有 roleId → 编辑模式

onMounted(async () => {
  await getTreeList()
  if (isEdit.value && roleId.value) {
    // 编辑模式：回填数据
    getRoleDetailEdit(roleId.value)
  }
})
```

| URL                    | 模式 | 说明                    |
| ---------------------- | ---- | ----------------------- |
| `/roleAdd`             | 新增 | `roleId` 为 `undefined` |
| `/roleAdd?roleId=4293` | 编辑 | `roleId` 为 `"4293"`    |

> **面试题：提交时怎么区分新增和编辑？**

```typescript
const createRole = async () => {
  if (isEdit.value) {
    // 编辑 → PUT 请求
    await updateRoleAPI({ ...roleForm.value, roleId: Number(roleId.value) })
  } else {
    // 新增 → POST 请求
    await createRoleUserAPI(roleForm.value)
  }
}
```

> **面试题：权限树是怎么实现的？**

使用 Element Plus 的 `el-tree` 组件，设置 `show-checkbox` 开启多选。

1. 调用接口获取完整的权限树数据（树形结构）
2. 渲染多棵树（园区管理、行车管理等每个模块一棵树）
3. 通过 `tree.getCheckedKeys()` 收集用户选中的权限 ID
4. 编辑模式下通过 `tree.setCheckedKeys([...])` 回填选中项

**收集权限代码：**

```typescript
const collectPerms = () => {
  roleForm.value.perms = []
  treeRef.value.forEach(tree => {
    roleForm.value.perms.push(tree.getCheckedKeys())
  })
}
```

## 三、项目亮点与难点

### 3.1 路由参数传递：params vs query

> **面试题：`route.params` 和 `route.query` 有什么区别？**

| 对比项   | params                     | query                |
| -------- | -------------------------- | -------------------- |
| URL 形式 | `/roleAdd/2`               | `/roleAdd?roleId=2`  |
| 获取方式 | `route.params.id`          | `route.query.roleId` |
| 路由配置 | 需要定义 `:id` 占位符      | 不需要额外配置       |
| 刷新页面 | 可能丢失（除非路由配置了） | 不会丢失             |
| 适用场景 | RESTful 风格               | 可选参数、筛选条件   |

**本项目选择 query**，因为编辑页和新增页复用同一路由，roleId 是可选参数。

### 3.2 TypeScript 类型约束

> **面试题：项目中怎么使用 TypeScript？**

```typescript
// 定义角色参数类型
interface RoleParams {
  roleName: string
  remark: string
  perms: number[][]
  roleId?: number // 编辑时必传，新增时可选
}

// API 函数参数类型约束
export const updateRoleAPI = (data: RoleParams) => {
  return request('/park/sys/role', 'PUT', data)
}
```

**好处：**

- 编辑器智能提示，减少拼写错误
- 编译期发现类型错误，避免运行时报错
- 接口参数类型一目了然，相当于文档

### 3.3 Axios 二次封装

> **面试题：你是怎么封装 Axios 的？**

1. **创建实例**：设置 `baseURL` 和 `timeout`
2. **请求拦截器**：统一注入 `token`
3. **响应拦截器**：统一处理错误（401 跳登录页、其他错误提示）
4. **封装通用方法**：支持传入 URL、方法、参数

### 3.4 Pinia 状态管理

> **面试题：为什么选择 Pinia 而不是 Vuex？**

| 对比项          | Pinia                     | Vuex                    |
| --------------- | ------------------------- | ----------------------- |
| 体积            | 更小                      | 较大                    |
| TypeScript 支持 | 原生支持                  | 需要额外配置            |
| 写法            | 支持 Composition API 风格 | 基于选项式 API          |
| mutations       | 不需要                    | 必须通过 mutations 修改 |
| 模块化          | 天然支持，每个 store 独立 | 需要 modules 配置       |

## 四、常见问题与解决方案

### 4.1 编辑角色时提示"入参字段 roleId 错误"

**原因：** `route.query.roleId` 返回的是**字符串**类型，而后端要求**数字**类型。

**解决：** 提交前用 `Number()` 转换。

```typescript
await updateRoleAPI({
  ...roleForm.value,
  roleId: Number(roleId.value) // "4293" → 4293
})
```

### 4.2 编辑角色发请求返回 400

**原因：** 编辑用的是 `createRoleUserAPI`（POST），但后端编辑接口要求 `PUT`。

**解决：** 新增用 POST，编辑用 PUT，调用不同的 API 函数。

```typescript
// 新增 - POST
export const createRoleUserAPI = (data: RoleParams) =>
  request('/park/sys/role', 'POST', data)
// 编辑 - PUT
export const updateRoleAPI = (data: RoleParams) =>
  request('/park/sys/role', 'PUT', data)
```

### 4.3 路由跳转报错 "No match for"

**原因：** 使用 `router.push({ name: 'RoleEdit', params: { roleId: '2' } })`，但路由表中没有定义 `name: 'RoleEdit'` 的路由。

**解决：** 改用已有路由 + query 参数：

```typescript
router.push(`/roleAdd?roleId=${roleId}`)
```

## 六、面试高频问题速记

### Vue 3 相关

1. **Composition API 和 Options API 的区别？**

   - Composition API 按功能组织代码，逻辑复用更方便（通过 composable 函数）
   - Options API 按选项分类（data/methods/computed），简单场景更直观

2. **ref 和 reactive 的区别？**

   - `ref` 用于基本类型，访问需要 `.value`，模板中自动解包
   - `reactive` 用于对象/数组，直接访问属性，不能重新赋值整个对象

3. **`<script setup>` 的优势？**
   - 更简洁，不需要 `return` 暴露变量
   - 更好的 TypeScript 类型推导
   - 编译时优化，性能更好

### 项目相关

1. **怎么做的分步表单？**

   - 用一个 `activeStep` 变量控制当前步骤
   - 通过 `v-if` / `v-show` 切换不同步骤的内容
   - 每一步切换前做校验（表单校验 / 权限校验）

2. **删除操作为什么要二次确认？**

   - 防止误操作，使用 `ElMessageBox.confirm` 弹出确认框
   - 确认后才调用删除接口，提升用户体验和数据安全性

3. **删除后列表怎么更新？**
   - 重新请求列表数据 `await getRolesList()`
   - 自动选中第一个角色，避免页面空白
